# PDF

The **PDF** functionality in billetsys makes it possible to turn ticket and report information into portable documents.

## Purpose

Not every support activity stays inside the live application. Teams often need documents they can share, archive, review offline, or include in meetings and customer communication. PDF export supports that need.

## Ticket export

One important use of PDF generation is ticket export. This allows a case to be captured as a self-contained document that reflects the current record of the ticket.

This is useful when a team needs to:

* Archive a case
* Share the case outside the live application
* Prepare documentation for review
* Preserve a support history snapshot

## Report export

PDF is also useful for analytical views. Reports inside billetsys can be turned into documents that help teams discuss trends, workload, and service performance without requiring everyone to open the reporting interface directly.

This makes the reporting feature easier to use in management and coordination contexts.

## Portable documentation

The value of PDF export is that it turns live application data into a portable format. Once exported, the information can be reviewed in a wider set of situations than the web application alone supports.

This helps billetsys serve both operational work and documentation needs.

## Role perspective

PDF export is especially useful for the roles that investigate, coordinate, review, and report on support activity. Ticket-focused roles benefit from document export for case handling, while oversight roles benefit from report export for analysis and communication.

## Why it matters

Without document export, important information would remain tied to the live interface. With PDF support, billetsys makes ticket and report information easier to circulate, preserve, and use in a broader operational context.
